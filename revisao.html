Revisando gráficos, escalas e método das diferenças finitas

A ideia deste texto é apresentar elementos para que seja possível a edição do código javascript e observar as mudanças neste documento, os resultados podem ser vistos nos quadros (canvas) que estão ao final do texto.

Eixos coordenados

Para criar os eixos adicionamos o elemento canvas ao html e identificamos ele no javascript e selecionamos o contexto para 2D. 

            
const can = document.getElementById("canvas1");
const ctx = canvas.getContext("2d");

Agora precisamos de duas retas para representar os eixos, para iniciar qualquer desenho dentro do canvas iniciamos pelo comando dentro do objeto de contexto (ctx), ctx.beginPath(), definir o estilo da linha e usar as funções moveTo(x0,y0) e lineTo(x1,y1) para desenhar o segmento de reta entre os pontos \(\left(x_0,y_0\right)\) e \(\left(x_1,y_1\right)\).

Uma forma de tornar as coisas mais automatizadas é obter as dimensõesdo canvas usando as funções que armazenam essas informações e fornecendo elas para variáveis, assim podemos escolher os pontos \(\left(x_0,y_0\right)\) e \(\left(x_1,y_1\right)\) sem estar se preocupando com os valores das dimensões do quadro: 

            
const can = document.getElementById("canvas1");
const ctx = canvas.getContext("2d");

const xp_max = ctx.canvas.width // largura em pixel
const yp_max = ctx.canvas.height // altura em pixel

// Posição em pixel onde os zeros de cada eixo irão se cruzar, divide por dois para o meio do quadro
const x0p = xp_max/2;
const y0p = yp_max/2;

ctx.beginPath();
ctx.strokeStyle = "black";
ctx.moveTo(0,y0p); ctx.lineTo(xp_max,y0p); // eixo x
ctx.moveTo(x0p,0); ctx.lineTo(x0p,yp_max); // eixo y
/*
comando para escrever as linhas no quadro,
sempre que terminar de escrever os elementos
que queira desenharr use-o para exibir
*/
ctx.stroke();

É possível posicionar o eixo y por exemplo num ponto que fique a 10% da dimensão do quadro basta trocar o fator \(1/2\) por \(10/100\) na definição da variável x0p: 

            
const x0p = 10/100*xp_xp_max; // tente outros valores inclusive com o eixo x

Para 20% basta usar o fator \(20/100\), e assim por diante.

Escalas nos eixos

Para escrevermos as escalas precisamos subdividir a reta em intervalos igualmente espaçados, vamos analisar o código abaixo: 

            
const can = document.getElementById("canvas2");
const ctx = can.getContext("2d");

const xp_max = ctx.canvas.width // largura em pixel
const yp_max = ctx.canvas.height // altura em pixel

// Posição em pixel onde os zeros de cada eixo irão se cruzar, divide por dois para o meio do quadro
const x0p = 2*xp_max/10;
const y0p = yp_max/2;

// Inicia o desenho
ctx.beginPath();
ctx.strokeStyle = "black";
    
// Eixos
ctx.moveTo(0,y0p); ctx.lineTo(xp_max,y0p); // eixo x
ctx.moveTo(x0p,0); ctx.lineTo(x0p,yp_max); // eixo y
    
// Escalas
const dim = 4 // dimensão das marcas nos eixos
    
// Marcas no eixo x
const Nx = 11; // Número de marcas no eixo x
let Dxp = xp_max/(Nx-1); // Espaçamento em pixel para colocar as marcas
for(let i=0;i<Nx;i++) {
xp = i*Dxp;
ctx.moveTo(xp,y0p-dim); ctx.lineTo(xp,y0p+dim);
}
ctx.stroke()

Observe que usamos o eixo y na posição igual a 20% da largura do quadro, para definirmos a escala em x precisamos definir os valores negativos e positivos de forma que 20% seja usado para os valores negativos e os outros 80% sejam usados nos valores positivos. Por exemplo se quisermos definir no eixo x uma escala dentro do intervalo \(\left[-5,5\right]\), para o zero do eixo x posicionado afastado apenas 20% da borda do canvas, podemos usar o intervalo \(\left[0,5\right]\) como sendo os 80%, os outros 20% pode ser calculado usando a regra de três \[ 80 \rightarrow 5-0 \] \[ 20 \rightarrow 0-x_{min}, \] com \(x\) o valor do início do intervalo nos números negativos. Pela regra de três temos \[ x_{min} = -5\frac{20}{80}=-\frac{5}{4}.\]

Dessa forma o intervalo para uma escala que respeita 20% dos valores do eixo x sendo os valores negativos e 80% dos valores sendo os positivos, temos o intervalo total dado por \(\left[-\frac{5}{4},5\right]\).

Podemos então definir as variáveis de interesse para os números da escala que iremos escrever ao lado das marcas: 

            
let xmin = -5/4; // início do intervalo;
let xmax = 5; // fim do intervalor;
let Deltax = xmax-xmin; // tamanho do intervalo na escala do eixo
let delta = Deltax/xp_max; // razão escala do eixo pela escala em pixel

Na seção anterior, aprendemos como posicionar as marcas no eixo x, temos então que calcular o valor da escala desejada a partir do valor em pixel usando a variável delta, que representaremos como \(\delta_x\) nesta equação, seja \(x_p\) o valor em pixel da localização da marca, o valor na escala do eixo será \[ x = \left(x_p-x_{0,\text{pixel}}\right)\cdot\delta_x. \]

Perceba que na equação acima o \(\delta_x\) possui dimensão da unidade da escala em \(x\) por pixel, como a subtração dentro dos aprênteses tem dimensão de pixel a unidade resultante deste produto é a unidade da escala em \(x\), consequentemente esse é o valor de \(x\) que está naquela posição em pixel. Caso posuíssemos o valor em \(x\) e necessitássemos encontrar a localização em pixel, bastaria inverter essa relação: \[ x_p = x_{0,\text{pixel}} + \frac{x}{\delta_x}. \]

O código completo no eixo x é portanto: 

            
const can = document.getElementById("canvas2");
const ctx = can.getContext("2d");

const xp_max = ctx.canvas.width // largura em pixel
const yp_max = ctx.canvas.height // altura em pixel

const x0p = 2*xp_max/10;
const y0p = yp_max/2;

// Inicia o desenho
ctx.beginPath();
ctx.strokeStyle = "black";
    
// Eixos
ctx.moveTo(0,y0p); ctx.lineTo(xp_max,y0p); // eixo x
ctx.moveTo(x0p,0); ctx.lineTo(x0p,yp_max); // eixo y
    
// Escalas
const dim = 4 // dimensão das marcas nos eixos
    
// Marcas no eixo x
const Nx = 11; // Número de marcas no eixo x
let Dxp = xp_max/(Nx-1); // Espaçamento em pixel para colocar as marcas
    
// Início do intervalo
let xmin = -5/4;
// Fim do intervalo
let xmax = 5;
// Tamanho do intervalo
let Deltax = xmax-xmin;
let deltax = Deltax/xp_max;
    
for(let i=0;i<Nx;i++) {
    let xp = i*Dxp; // posição x em pixel
    let x = (xp-x0p)*deltax; // posição x na escala do eixo
    ctx.moveTo(xp,y0p-dim); ctx.lineTo(xp,y0p+dim); // desenha a linha
    ctx.fillText(x.toFixed(3),xp+4,y0p+9);
}
ctx.stroke()

Perceba que a equação para o eixo \(y\) tem uma diferença pois o primeiro pixel no eixo vertical começa de cima e vai crescendo pra baixo, as equaçõe de convenção são portanto para a posição na escala, sabendo a posição em pixel e veice-versa: \[ y = \left(y_{0,\text{pixel}}-y_p\right)\cdot\delta_y, \] \[ y_p = y_{0,\text{pixel}} - \frac{y}{\delta_y}. \]

Diferenças finitas

Nosso objetivo com esse método é calcular as variáveis de uma dinâmica a partir da adição de pequenas diferenças calculadas conforme as equações do movimento do sistema estudado. No movimento bidimensional uma possibilidade de efetuarmos isso é calculando as sequintes equações, para cada corpo em movimento: \[ x_{i+1} = x_i +\Delta x; \] \[ v_{x,i+1} = v_{x,i} +\Delta v_x; \] \[ y_{i+1} = y_i +\Delta y; \] \[ v_{y,i+1} = v_{y,i} +\Delta v_y. \]

Essa coleção de \(\Delta\)'s geralmente são funções de todas as coordenadas envolvidas no sistema, isto é, dependem das posições e velocidades envolvidas no problema. Ao analisarmos um problema que envolva a interação entre dois corpos isso fica mais explícito, vamos escrever as equações de interação gravitacional entre dois corpos de massa \(m_1\) e \(m_2\) respectivamente, podemos escrever as equações do movimento dos dois corpos, por simplicidade vamos imaginar apenas o movimento bidimensional no plano em que os corpos desenvolvem seu movimento um em torno do outro, veja imagem

As equações do movimento serão: \[m_1\vec a_1 = -G\frac{m_1 m_2}{r^2}\hat r,\] \[m_2\vec a_2 = G\frac{m_1 m_2}{r^2}\hat r.\]

Nestas equações temos que prestar atenção no sentido do vetor \(\mathbf r\) que sai de \(m_2\) e encontra \(m_1\), o r com chapéu (\(\hat r\)) indica essa direção e sentido para a força, como \(m_1\) sofre uma atração de \(m_2\), o sinal negativo deve ser usado pra representar o vetor força agindo em \(m_1\), dado que essa força está em sentido contrário ao vetor \(\mathbf r\). Já a força que atua no corpo \(m_2\) é exatamente no sentido do vetor \(\mathbf r\), e o módulo delas é idêntico por se tratar de um par de ação e reação.

Agora escrevemos os elementos dessas equações usando as coordenadas cartesianas, primeiro o módulo do vetor \(\mathbf r\), \[ r = \sqrt{\left(x_1-x_2\right)^2 + \left(y_1-y_2\right)^2}, \] uma vez que o vetor \(\mathbf r\) é dado em coordenadas cartesianas por \[ \mathbf r = \left(x_1-x_2\right)\hat i + \left(y_1-y_2\right)\hat j, \] com \(\hat i\) e \(\hat j\) indicando a direção e sentido positivo dos eixos \(x\) e \(y\) respectivamente.

O versor que indica a direção do vetor \(\mathbf r\) pode ser escrito por: \[ \hat r = \frac{\mathbf r}{r}. \]

Com esses elementos vamos separar as duas equações vetoriais acima em quatro equações nas coordenadas \(x\) e \(y\) de cada aceleração: \[ a_{1,x} = -G\frac{m_2}{r^3}(x_1-x_2); \] \[ a_{1,y} = -G\frac{m_2}{r^3}(y_1-y_2); \] \[ a_{2,x} = G\frac{m_1}{r^3}(x_1-x_2); \] \[ a_{2,y} = G\frac{m_1}{r^3}(y_1-y_2). \]

Já vimos anteriormente que a equação da aceleração gera a equação de diferenças finitas da velocidade, e a equação e a equação da velocidade gera a diferença finita da posição, por simplicidade o fator comum das equações acima usaremos uma notação mais compacta definindo \[ f_x = G\frac{\left(x_1-x_2\right)}{r^3}, \] \[ f_y = G\frac{\left(y_1-y_2\right)}{r^3}, \] com \[ r^3 = \left(\sqrt{\left(x_1-x_2\right)^2 + \left(y_1-y_2\right)^2}\right)^3. \]

O conjunto de equações de diferenças finitas fica portanto, para velocidades \[ v_{x,1|i+1} = v_{x,1|i} -m_2f_x\cdot\Delta t \] \[ v_{x,2|i+1} = v_{x,2|i} +m_1f_x\cdot\Delta t \] \[ v_{y,1|i+1} = v_{y,1|i} -m_2f_y\cdot\Delta t \] \[ v_{y,2|i+1} = v_{y,2|i} +m_1f_y\cdot\Delta t \] e posições \[ x_{1,i+1} = x_{1,i} + v_{x,1|i}\cdot \Delta t \] \[ x_{2,i+1} = x_{2,i} + v_{x,2|i}\cdot \Delta t \] \[ y_{1,i+1} = y_{1,i} + v_{y,1|i}\cdot \Delta t \] \[ y_{2,i+1} = y_{2,i} + v_{y,2|i}\cdot \Delta t. \]

Supondo que fóssemos simular o movimento ao longo do eixo \(z\) pra descrever o movimentos de forma tridimensional precisaríamos ter mais uma equação para as posições e velocidades nesse eixo, para cada corpo em movimento, seriam portanto mais quatro equações, totalizando 12 equações para descrever o movimento tridimensional de dois corpos em interação, tente responder para o caso de três corpos interagindo em três dimensões.

Simulando

Vamos usar o canvas para observar a órbita desenvolvida por dois corpos com interação gravitacional, o Sol e a Terra por exemplo, primeiro vamos ajustar o canvas para ter as dimensões na escala dos eixos com a representração dessa órbita, uma rápida pesquisa é possível descobrir a distância média da Terra ao Sol (\(1,476\times 10^8km\)) e a velocidade média (\(29.800m/s\)) desde planeta em torno da nossa estrela, isso é o suficiente para sabermos qual escala devemos usar no canvas.

Podemos portanto escolher o intervalo nos eixos do canvas como indo de \(\left[-2,2\right]\) nos dois eixos, dada posição ali representada deve ser multiplicada por \(10^{11}\), uma vez que a posição \(1,476\) neste gráfico irá representar a posição inicial da Terra, com o Sol no ponto \((0,0)\) do gráfico. Nessa escala a velocidade inicial que usaremos terá que ser \(2,98\times 10^{-7}\) uma vez que cada unidade neste gráfico equivale ao deslocamento de \(10^{11}m=10^8km\) na escala real. Usando esses valores qualquer valor que escolhermos para \(\Delta t\) estará representado em segundos.

Para facilitar a simulação escolhemos a constante gravitacional \(G = 1\), dessa forma não precisamos incluí-la nos cálculos, caso necessitemos escrever as posições com as coordenadas reais cuidamos para que essa constante seja reinserida por meio de uma simples multiplicação ao resultado. Aplicaremos essa mesma técnica para as velocidades e posições iniciais.

Dessa forma podemos definir as condições iniciais da seguinte forma: 

            
const m1 = 1; // Massa do Sol
const m2 = 0.000003002513826043238; // massa da Terra dividida pela massa do Sol
